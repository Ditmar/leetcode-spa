# AuthModal

The `AuthModal` component provides a unified, highly accessible interface for user authentication. It seamlessly supports both **Sign In** and **Sign Up** flows within a responsive Material UI `Dialog`, following strict Component-Driven Development (CDD) principles.

## Usage

To use the `AuthModal`, you need to manage its `isOpen` state from the parent component. 

```tsx
import { useState } from 'react';
import { Button } from '@mui/material';
import { AuthModal } from './AuthModal';
import { AuthMode } from './AuthModal.types';

export const App = () => {
  const [isModalOpen, setIsModalOpen] = useState(false);

  return (
    <>
      <Button onClick={() => setIsModalOpen(true)} variant="contained">
        Login
      </Button>

      <AuthModal 
        isOpen={isModalOpen} 
        onClose={() => setIsModalOpen(false)} 
        initialMode={AuthMode.SIGN_IN} 
      />
    </>
  );
};
```

## Props Reference

The component accepts the following properties to control its behavior and initial state:

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| **`isOpen`** | `boolean` | `false` | **Required.** Controls the visibility of the modal dialog. |
| **`onClose`** | `() => void` | `undefined` | **Required.** Callback fired when the component requests to be closed (e.g., backdrop click, escape key, or close button click). |
| **`initialMode`** | `AuthMode` | `AuthMode.SIGN_IN` | Optional. Determines which form (Sign In or Sign Up) is rendered when the modal initially opens. |

## Dual-Mode Behavior

The `AuthModal` encapsulates two distinct user flows without requiring a page navigation or modal unmount, improving perceived performance.

### Sign In Mode
- **Fields:** Email, Password.
- **Features:** Includes a "Remember me" checkbox and a "Forgot password?" hyperlink.
- **Action:** Authenticates returning users.

### Sign Up Mode
- **Fields:** Full Name, Email, Password, Confirm Password.
- **Features:** Focuses purely on account creation. Hides the "Remember me" and "Forgot password?" elements to reduce cognitive load.
- **Action:** Registers new users.

*State persistence:* Toggling between modes triggers a form reset, clearing any previous inputs and validation errors to ensure a clean state.

## Form Validation

Form state and validation are handled robustly via `react-hook-form` and mapped to a strict schema using `zod` (`signInSchema` and `signUpSchema`). Errors are displayed in real-time (`mode: 'onTouched'`) via MUI's `helperText`.

**Validation Rules:**
* **Full Name (Sign Up only):** Must be at least 2 characters long.
* **Email:** Must be a non-empty string matching a valid email format.
* **Password:** * Minimum of 8 characters.
  * Must contain at least one uppercase letter (`/[A-Z]/`).
  * Must contain at least one number (`/[0-9]/`).
* **Confirm Password (Sign Up only):** Must perfectly match the Password field.
* **Remember Me (Sign In only):** Optional boolean.

## Accessibility (a11y)

Accessibility is a primary concern for authentication flows. This component implements the following WAI-ARIA standards:

* **Focus Management:** Utilizes MUI's native `Dialog` focus trap. Focus is automatically shifted to the first interactive element upon opening and returned to the triggering element upon closing.
* **Roles & Attributes:** * The container has `role="dialog"` and `aria-modal="true"`.
  * The title is linked via `aria-labelledby="auth-modal-title"`.
* **Keyboard Navigation:** * Pressing `Escape` triggers the `onClose` callback.
  * `Tab` navigation cycles logically through inputs, checkboxes, and buttons.
* **Touch Targets:** The top-right close `IconButton` strictly adheres to mobile guidelines, featuring a minimum `44x44px` touch target area.
* **Social Buttons:** Social authentication buttons (Google, GitHub) include explicit `aria-label` attributes to ensure screen readers announce their specific function.

## Responsive Behavior

The modal adapts fluidly across device sizes, controlled via MUI's `useMediaQuery`:

* **Mobile (`< 600px`, `sm` down):** * Renders in `fullScreen` mode.
  * Border radius is removed (`0px`) to blend seamlessly with the device viewport.
* **Tablet & Desktop (`>= 600px`, `sm` up):** * Constrained to a maximum width of `480px` (`maxWidth="xs"`).
  * Centered vertically and horizontally with a subtle backdrop shadow.
  * Features a rounded aesthetic with `border-radius: 12px`.
