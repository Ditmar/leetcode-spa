# Label

The `Label` component renders an accessible form label that programmatically and visually associates with its input control. It is a **pure presentational atom** built on top of MUI `InputLabel`.

---

## Usage

```tsx
import Label from '@/components/Label';

<Label htmlFor="email-input" required>
  Email address
</Label>
<input id="email-input" type="email" required />
```

---

## Features

| Feature | Prop | Behaviour |
|---|---|---|
| Required indicator | `required` | Appends a red `*` glyph; sets `aria-required="true"` on the `<label>` |
| Optional hint | `optional` | Appends a grey italic `(optional)` span |
| Error state | `error` | Text rendered in `theme.palette.error.main` |
| Disabled state | `disabled` | Reduced opacity via MUI `Mui-disabled` class |
| Contextual tooltip | `tooltip` | Renders an info icon button; tooltip opens on hover, focus, or tap |

> **Note:** `required` and `optional` are mutually exclusive. If both are set, `required` takes precedence and `optional` is suppressed.

---

## Accessibility

- The `htmlFor` prop **must** match the `id` of the associated input — this creates the programmatic association required by WCAG 1.3.1 (Info and Relationships).
- When `required` is true, `aria-required="true"` is set on the `<label>` so screen readers announce the constraint even before the user interacts with the field.
- The `*` glyph is wrapped in `aria-hidden="true"` to prevent double-announcement (the `aria-required` attribute is the canonical signal).
- The tooltip icon has an accessible `aria-label` (`"More information"`) and is reachable by keyboard (`Tab`) and touch (`enterTouchDelay={0}`).
- All text sizing uses `rem`-based MUI typography tokens, so the component respects OS-level font scaling (tested at 200%).
- No content truncates at a 320 px viewport width.
