# Label

An **accessible**, **responsive**, and **theme-aware** form label component
built on top of MUI `InputLabel`.

The component supports common form-label states such as:

- required
- optional
- error
- disabled
- tooltip messaging

It is designed as a lightweight presentational atom that integrates seamlessly
with form controls and accessibility workflows.

---

# Features

- Accessible form labeling
- Fully responsive layout
- Theme-aware styling
- Optional tooltip support
- Required and optional indicators
- Error and disabled states
- Keyboard accessible interactions
- Compatible with MUI form controls

---

# Overview

The `Label` component visually and programmatically associates itself
with form controls through the `htmlFor` prop.

| State | Description |
| --- | --- |
| `required` | Displays required indicator and accessibility metadata |
| `optional` | Displays optional helper text |
| `error` | Applies error styling from the theme |
| `disabled` | Applies disabled visual state |
| `tooltip` | Displays contextual helper information |

---

# Installation

```bash
yarn add @mui/material @emotion/react @emotion/styled
```

---

# Quick Start

```tsx
import Label from './Label';

export default function Example() {
  return (
    <>
      <Label
        htmlFor="email-input"
        required
      >
        Email address
      </Label>

      <input
        id="email-input"
        type="email"
        required
      />
    </>
  );
}
```

# Prop Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `htmlFor` | `string` | — | Required. Associates label with input id |
| `children` | `React.ReactNode` | — | Label content |
| `required` | `boolean` | `false` | Displays required indicator |
| `optional` | `boolean` | `false` | Displays optional helper text |
| `error` | `boolean` | `false` | Applies error visual state |
| `disabled` | `boolean` | `false` | Applies disabled state |
| `tooltip` | `string` | — | Tooltip helper content |

---

# Accessibility

- `htmlFor` associates the label with its target form control
- Required fields propagate `aria-required="true"`
- Required indicators use `aria-hidden="true"`
- Tooltip buttons expose accessible labels
- Tooltip interactions support keyboard navigation
- Accessibility behaviour validated with Testing Library
- Wrapped labels remain readable in narrow layouts

---

# Responsive Behaviour

| Viewport | Behaviour |
| --- | --- |
| `320px` | Wrapped label text without clipping |
| `375px` | Comfortable spacing and inline layout |
| `768px` | Stable alignment with form controls |
| `1024px+` | Inline layout within wider forms |

All spacing and typography values are derived from the MUI theme.

No external CSS is required.

- MUI `InputLabel`

- Theme palette values

# Troubleshooting

## If the Storybook Docs page does not appear

Verify the following:

### 1. Story Export Exists

Ensure all stories are exported correctly in:

```txt
Label.stories.tsx
```

Example:

```tsx
export const Default = {};
```

---

### 2. Storybook Main Configuration

Ensure `.storybook/main.ts` contains:

```ts
stories: [
  '../src/**/*.mdx',
  '../src/**/*.stories.@(js|jsx|ts|tsx)',
];
```

---

### 3. Build Validation

Run:

```bash
yarn build-storybook
```

If this command fails, the PR preview deployment will fail as well.

---

### 4. Linux Filename Validation

Ensure filenames match exactly:

```txt
Label.docs.mdx
```

Avoid mismatched casing like:

```txt
label.docs.mdx
```

---

### 5. Verify Storybook Deploy

Check GitHub Actions or Chromatic build logs for:

- MDX parsing errors
- Missing exports
- Invalid imports
- Story loading failures
