Full-page organism that renders a searchable, filterable problem browser built with **Material UI**.

## Architecture

### Data injection via props

The component accepts a `problems` prop so the data source is fully decoupled:

```tsx
// Default — uses the static dataset
<ProblemsPage onSelectProblem={fn} onNavigateToCode={fn} />

// API data — fetched by a parent, passed as prop
<ProblemsPage problems={apiProblems} onSelectProblem={fn} onNavigateToCode={fn} />

// Tests — inject a fixture directly, no module mocks needed
<ProblemsPage problems={FIXTURE} onSelectProblem={fn} onNavigateToCode={fn} />
```

### Hook — `useProblemsPage({ problems })`

All state and derived data live in the hook. It receives `problems` as a parameter and exposes:

```ts
const {
  filteredProblems,
  solvedCount,
  totalCount,
  tagOptions,
  filterState,
  handleSearchChange,
  handleDifficultyChange,
  handleStatusChange,
  handleTagChange,
  handleClearFilters,
  drawerOpen,
  handleDrawerOpen,
  handleDrawerClose,
} = useProblemsPage({ problems });
```

### Status filter — `STATUS_FILTER_MAP`

Status matching uses a Record instead of scattered string comparisons.
Adding a new status only requires updating one object:

```ts
const STATUS_FILTER_MAP: Record<Exclude<StatusFilter, 'All'>, ProblemStatus> = {
  Solved: 'solved',
  Attempted: 'attempted',
  Unsolved: 'unsolved',
};
```

### Tag sentinel — `ALL_TAGS_SENTINEL`

The "no tag filter" state is handled in a single `matchesTagFilter` helper.
The sentinel never appears in the component template.

```ts
function matchesTagFilter(p: Problem, tagFilter: TagFilter): boolean {
  if (tagFilter === ALL_TAGS_SENTINEL) return true;
  return p.tags.includes(tagFilter);
}
```

## Filtering logic

All four filters are applied with `&&`. A problem must match every active criterion.

| Filter     | Default | Options                                       |
| ---------- | ------- | --------------------------------------------- |
| Search     | `""`    | Free-text substring (title), debounced 300 ms |
| Difficulty | `"All"` | All · Easy · Medium · Hard                    |
| Status     | `"All"` | All · Solved · Attempted · Unsolved           |
| Tag        | `"All"` | All · _derived from dataset_                  |

## Visual design

| Difficulty | Chip colour  | Status    | Icon                          |
| ---------- | ------------ | --------- | ----------------------------- |
| Easy       | `success` 🟢 | solved    | `CheckCircleOutline` (green)  |
| Medium     | `warning` 🟡 | attempted | `RemoveCircleOutline` (amber) |
| Hard       | `error` 🔴   | unsolved  | `RadioButtonUnchecked` (grey) |

Solved rows have a **4 px left border** in `theme.palette.success.main`.
