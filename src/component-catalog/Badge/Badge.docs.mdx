# Badge

The Badge component is used to display a small visual indicator that helps identify status, category, or short contextual information inside the interface.

This component supports two display modes:

- **Standalone**: used as a text badge to represent categories or visual states.
- **Attached**: used as a Material UI badge attached to another element, for example to show counts, notifications, or a dot indicator.

## What this component does

The purpose of this component is to make small pieces of information easier to notice without taking too much space in the layout.

It can be used to:

- show category labels
- represent status or semantic meaning
- display notification counts
- show overflow values such as `99+`
- display a dot indicator for unread or active states

Because this is a compact component, it should remain readable, visually clear, and consistent across different screen sizes.

## Variants

For the standalone version, the component includes these visual variants:

- **Default**
- **Secondary**
- **Outline**
- **Destructive**
- **Success**
- **Info**
- **Warning**

Each variant changes the visual appearance of the badge depending on its intended meaning.

For the attached version, the component also supports:

- numeric content
- max overflow
- dot variant
- multiple Material UI color variants
- configurable anchor positions

## Example

Standalone badge:

```tsx
<Badge display="standalone" badgeContent="Success" tone="success" />
```
