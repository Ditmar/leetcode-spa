# NavigationMenu

A **fully-responsive**, **accessible** navigation component built on
[MUI `AppBar`](https://mui.com/material-ui/react-app-bar/),
[`Toolbar`](https://mui.com/material-ui/react-toolbar/), and
[`Drawer`](https://mui.com/material-ui/react-drawer/) with **keyboard navigation**
and **mobile-friendly** dropdown support.

---

## Overview

The `NavigationMenu` component renders a primary navigation structure that adapts
to screen size:

| Viewport  | Layout                                         |
| --------- | ---------------------------------------------- |
| Desktop   | Horizontal nav bar with optional flyout panels |
| Mobile    | Hamburger icon → Drawer with accordion nav     |

**Key features:**
- ✓ Horizontal nav bar on desktop (≥ 768px)
- ✓ Hamburger drawer on mobile (< 768px)
- ✓ Active item highlighted via `aria-current="page"`
- ✓ Optional logo/brand slot
- ✓ Keyboard navigation (`Tab`, `Arrow keys`, `Escape`)
- ✓ Touch targets ≥ 44×44px on mobile
- ✓ Fully-responsive: 320px → 1440px tested

## Quick Start

### Basic Navigation (Desktop + Mobile)

```tsx
import { NavigationMenu } from '@/component-catalog/NavigationMenu';

const navSections = [
  {
    id: 'main',
    items: [
      { id: 'premium', label: 'Premium', href: '/premium' },
      { id: 'explore', label: 'Explore', href: '/explore' },
      { id: 'problems', label: 'Problems', href: '/problems' },
    ],
  },
];

export function App() {
  const [currentPath, setCurrentPath] = React.useState('/premium');

  const handleNavigation = (item: NavItem) => {
    if (item.href) {
      setCurrentPath(item.href);
      // Implement your routing logic here
    }
  };

  return (
    <NavigationMenu
      navSections={navSections}
      currentPath={currentPath}
      onItemClick={handleNavigation}
    />
  );
}
```

## Accessibility

**Screen reader support:**
- Navigation container marked with `role="navigation"` and `aria-label`
- Active item annotated with `aria-current="page"`
- Disabled items carry `aria-disabled="true"`
- Hamburger button has descriptive `aria-label="Toggle menu"`

**Keyboard navigation:**
- `Tab` / `Shift+Tab` → Move focus between menu items
- `Arrow keys` → Navigate within nav groups (future enhancement)
- `Escape` → Close mobile drawer
- `Enter` / `Space` → Activate menu item or toggle drawer

**Touch targets:**
- All interactive elements ≥ 44×44 px on mobile (WCAG 2.5.5 compliance)
- Desktop buttons maintain minimum 32×32 px

**Testing:**
- Component passes `axe-core` accessibility audit
- Supports JAWS, NVDA, and VoiceOver screen readers

### navigationMenuTokens Structure

The `navigationMenuTokens` object contains all styling, spacing, and sizing configuration:

| Category | Properties | Usage |
|----------|-----------|-------|
| **colors** | `background`, `border`, `text` | AppBar, drawer, and text colors |
| **sizes** | `appBarHeight` (3 variants), `logoHeight`, `drawerWidth`, `megaPanel` | Responsive dimensions |
| **minTouchTarget** | `5.5` | WCAG touch target minimum (→ 44px with theme.spacing) |
| **spacing** | `itemPaddingY/X`, `toolbarGap`, `iconMargin`, `logoMargin`, etc | Layout spacing |
| **signInButton** | `fontWeight`, `height`, `paddingX`, `fontSize`, `borderRadius` | Sign-in button styling |
| **layout** | `appBar`, `toolbar`, `iconContainer`, `logoContainer`, `desktopNav`, `mobileNav`, `drawer`, `scrollbar` | Flexbox & position configs |
| **listItem** | `position`, `fontSize`, `fontWeight`, `hover`, `active`, `disabled` | Menu item states |
| **accordion** | `boxShadow`, `expanded`, `summaryButton`, `summaryFocused` | Mobile accordion styling |
| **megaPanel** | `popperOffset`, `borderRadius` | Flyout panel positioning |
| **opacity** | `accordion`, `border` | Transparency values |
| **borders** | `width` | Border thickness |
| **scroll** | `hideThreshold` | Scroll hide trigger (100px) |

### Size Token Values

All size values are expressed in **rem** (relative em units) for consistency with the project's design system. This allows for scalable, accessible sizing that respects user font-size preferences.

| Token | Value | Equivalent |
|-------|-------|------------|
| **appBarHeight.small** | `'3rem'` | 48px |
| **appBarHeight.medium** | `'3.5625rem'` | 57px |
| **appBarHeight.large** | `'4rem'` | 64px |
| **logoHeight** | `'1.5rem'` | 24px |
| **drawerWidth** | `'16rem'` | 256px |
| **megaPanel.minWidth** | `'17.5rem'` | 280px |
| **megaPanel.maxWidth** | `'25rem'` | 400px |

**Conversion formula:** `px ÷ 16 = rem` (assuming base font-size of 16px)

### Token Unit Consistency

The `navigationMenuTokens` object uses standardized unit representations to avoid ambiguity:

| Unit Type | Usage | Examples |
|-----------|-------|----------|
| **String rem** | Direct CSS values (no conversion) | `fontSize: '0.875rem'`, `height: '1.75rem'`, `logoHeight: '1.5rem'` |
| **Number multiplier** | Values passed to `theme.spacing()` | `paddingX: 2`, `letterSpacing: 0.05`, `borderRadius: 0.5` |
| **Pure number** | Unitless CSS properties | `fontWeight: 600` |

**Example from signInButton:**
```typescript
signInButton: {
  fontWeight: 600,           // Pure number (unitless)
  height: '1.75rem',         // String rem (28px)
  paddingX: 2,               // Multiplier → theme.spacing(2) = 16px
  fontSize: '0.875rem',      // String rem
  letterSpacing: 0.05,       // Multiplier → theme.spacing(0.05) for conversion
  borderRadius: 0.5,         // Multiplier → theme.spacing(0.5) = 4px
}
```

**Usage in NavigationMenu.tsx:**
```typescript
const signInButtonSx = {
  ...navigationMenuTokens.signInButton,
  letterSpacing: theme.spacing(navigationMenuTokens.signInButton.letterSpacing),
  borderRadius: theme.spacing(navigationMenuTokens.signInButton.borderRadius),
  // ... other properties
};
```

Example usage in styles:
```typescript
const StyledAppBar = styled(AppBar)(({ theme }) => ({
  height: navigationMenuTokens.sizes.appBarHeight.medium, // '3.5625rem'
  backgroundColor: navigationMenuTokens.colors.background,
}));
```

#### **4. NavigationMenu.utils.ts (Configuration Helpers)**

// ✅ From theme.ts
import { navigationMenuTokens } from '../../style-library/theme/theme';
import type { Theme } from '@mui/material';

// Usage in utility function:
function getSizeConfig(size: NavigationMenuSize = 'medium', theme: Theme) {
  const sizeConfigs = {
    small: {
      appBarHeight: navigationMenuTokens.sizes.appBarHeight.small,
      itemPadding: `${theme.spacing(navigationMenuTokens.spacing.itemPaddingY.small)} ${theme.spacing(navigationMenuTokens.spacing.itemPaddingX.small)}`,
      itemFontSize: theme.typography.caption.fontSize,
      toolbarGap: navigationMenuTokens.spacing.toolbarGap.small,
    },
    medium: {
      appBarHeight: navigationMenuTokens.sizes.appBarHeight.medium,
      itemPadding: `${theme.spacing(navigationMenuTokens.spacing.itemPaddingY.medium)} ${theme.spacing(navigationMenuTokens.spacing.itemPaddingX.medium)}`,
      itemFontSize: theme.typography.body2.fontSize,
      toolbarGap: navigationMenuTokens.spacing.toolbarGap.medium,
    },
    large: {
      appBarHeight: navigationMenuTokens.sizes.appBarHeight.large,
      itemPadding: `${theme.spacing(navigationMenuTokens.spacing.itemPaddingY.large)} ${theme.spacing(navigationMenuTokens.spacing.itemPaddingX.large)}`,
      itemFontSize: theme.typography.body1.fontSize,
      toolbarGap: navigationMenuTokens.spacing.toolbarGap.large,
    },
  };
  return sizeConfigs[size];
}
