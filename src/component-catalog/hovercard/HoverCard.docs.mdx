# HoverCard Component

The **HoverCard** component displays a rich floating preview card anchored to a
trigger element. It implements the **'Tooltips & Hover Cards'** pattern from the
Figma design system and follows **Component-Driven Development (CDD)** principles
using MUI primitives (`Popper`, `Paper`, `Fade`, `ClickAwayListener`).

---

## Features

- Built with `@mui/material/Popper` and `Paper`
- Opens on `mouseenter` / `focus` (desktop) and `click` (touch devices)
- Configurable open and close delay (default: 100 ms / 100 ms)
- Viewport-clamped via `@popperjs/core` modifiers
- Smooth `Fade` transition on open/close
- `ClickAwayListener` closes the card on outside click/tap
- Fully responsive — fluid width at 320 px and 375 px, capped at `maxWidth` on wider screens
- Figma-faithful ghost trigger button (`HoverTriggerButton`) included

## Interaction model

| Device          | Open gesture                     | Close gesture                          |
| --------------- | -------------------------------- | -------------------------------------- |
| Desktop / mouse | `mouseenter` or keyboard `focus` | `mouseleave`, `blur`, or click outside |

---

## Responsive behaviour

| Viewport | Card behaviour                          |
| -------- | --------------------------------------- |
| 320 px   | Fluid: `calc(100vw - theme.spacing(4))` |
| 375 px   | Fluid (same rule)                       |
| 768 px   | Capped at `maxWidth` (default 320 px)   |
| 1024 px  | Capped at `maxWidth` (default 320 px)   |

---

## Accessibility

- The card carries `role='tooltip'` for screen-reader compatibility.
- Keyboard users can open via `Tab` (focus) and close by navigating away.
- Passes axe / WCAG 2.1 AA checks.
- `HoverTriggerButton` includes a `:focus-visible` outline using `theme.palette.primary.main`.
