# ProblemList

A **scrollable sidebar** organism that renders a list of coding problems with status
indicators, difficulty labels, and acceptance rates.
Built with **Material UI** and the project's centralised design-token system.

---

## Overview

| Feature | Detail |
| ------- | ------ |
| Status icons | Solved → green `CheckCircleOutline`, Attempted → amber `RemoveCircleOutline`, Unsolved → grey `RadioButtonUnchecked` |
| Difficulty | Coloured plain text — Easy `#00C950`, Medium `#F0B100`, Hard `#FB2C36` |
| Selection | Active item: `3.52 px` blue left border + `#EFF6FF` background |
| Responsive | Desktop ≥ 768 px: fixed sidebar. Mobile < 768 px: Drawer toggled by a Fab |
| Accessibility | `aria-current`, icon `aria-label`, keyboard navigation |
| Font | **Inter** (500 / 600) — registered in `theme.ts` `@font-face` |

---

## Quick Start

```tsx
import { ProblemList } from '@/pages/ProblemList';
import type { Problem } from '@/pages/ProblemList/ProblemList.types';

const problems: Problem[] = [
  { id: 1, title: 'Two Sum', difficulty: 'Easy', acceptanceRate: 49.2, status: 'solved' },
  { id: 2, title: 'Add Two Numbers', difficulty: 'Medium', acceptanceRate: 41.8, status: 'attempted' },
];

export function App() {
  const [selected, setSelected] = React.useState(1);
  return (
    <ProblemList
      problems={problems}
      selectedProblemId={selected}
      onSelectProblem={setSelected}
    />
  );
}
```

## Prop Reference

### ProblemListProps

| Prop | Type | Required | Description |
| ---- | ---- | :------: | ----------- |
| `problems` | `Problem[]` | ✓ | Array of problem objects to display |
| `selectedProblemId` | `number` | ✓ | ID of the currently highlighted problem |
| `onSelectProblem` | `(id: number) => void` | ✓ | Callback fired when a row is clicked |

### Problem

| Field | Type | Description |
| ----- | ---- | ----------- |
| `id` | `number` | Unique identifier |
| `title` | `string` | Display name |
| `difficulty` | `'Easy' \| 'Medium' \| 'Hard'` | Difficulty level |
| `acceptanceRate` | `number` | Percentage value (e.g. `49.2`) |
| `status` | `'solved' \| 'attempted' \| 'unsolved'` | User progress |

## Accessibility

- Every `ListItemButton` has a descriptive `aria-label` including title, difficulty, and acceptance rate.
- The selected item receives `aria-current="true"`.
- Status icons carry `aria-label`: `"Solved"`, `"Attempted"`, `"Unsolved"`.
- All items are keyboard-focusable and activatable via `Enter` / `Space`.
- Mobile Fab: `aria-label="Open problems list"`.
