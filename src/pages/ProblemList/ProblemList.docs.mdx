## Overview

`ProblemList` renders a fixed sidebar column with a scrollable list of coding problems. Each row displays:

- A **status icon** (solved / attempted / unsolved)
- The **sequential number** and **title** of the problem
- A **colored difficulty tag** (Easy / Medium / Hard)
- The **acceptance rate** percentage
- A **vertical blue bar** on the currently selected item

The component is **fully controlled**: it holds no selection state of its own. The parent component decides which problem is selected via the `selectedProblemId` prop and reacts to changes through the `onSelectProblem` callback.

| `difficulty` | `ProblemDifficulty` | Difficulty level. Determines the color of the difficulty tag text |
| `acceptanceRate` | `number` | Problem acceptance rate percentage. Displayed as-is with the `%` symbol |
| `status` | `ProblemStatus` | The user's status on this problem. Determines the icon and its color |

## Basic Usage

### Minimal example

```tsx
import { ProblemList } from './ProblemList';

const problems = [
  {
    id: 1,
    idNumber: 1,
    title: 'Two Sum',
    difficulty: 'Easy',
    acceptanceRate: 49.2,
    status: 'solved',
  },
];

function App() {
  const [selectedId, setSelectedId] = useState<number>(1);

  return (
    <ProblemList
      problems={problems}
      selectedProblemId={selectedId}
      onSelectProblem={(id) => setSelectedId(id)}
    />
  );
}
```

### With external state (Zustand / Redux)

The component is fully controlled, so it integrates seamlessly with any store:

```tsx
// With Zustand
const selectedId = useStore((s) => s.selectedProblemId);
const setSelected = useStore((s) => s.setSelectedProblem);

<ProblemList
  problems={problems}
  selectedProblemId={selectedId}
  onSelectProblem={setSelected}
/>
```

### With an empty list

When `problems` is an empty array, the component renders the "Problems" header and an empty scroll area without errors:

```tsx
<ProblemList
  problems={[]}
  selectedProblemId={undefined}
  onSelectProblem={() => {}}
/>
```

The most important style function. It receives a boolean and returns the style object for each row, including the CSS pseudo-elements for the blue bar.

```typescript
  // Fixed dimensions: ensures ALL items have the same height
  height: '62.15px',
  minHeight: '62.15px',
  maxHeight: '62.15px',

  // Uniform padding in all states
  paddingTop: '12px',
  paddingBottom: '12px',
  paddingLeft: '12px',
  paddingRight: '12px',

  '&::before': {
    content: '""',
    position: 'absolute',
    left: 0,
    top: 0,
    bottom: 0,
    width: '3.52px',
    backgroundColor: isSelected ? '#2B7FFF' : 'transparent',
  },

  // Hover only when not selected
  '&:hover': {
    backgroundColor: isSelected ? '#EFF6FF' : '#f9fafb',

});
```

> **Note on `Record<string, unknown>`:** This type is used instead of `SxProps<Theme>` to avoid strict compilation conflicts with nested pseudo-element properties inside MUI `sx` objects built dynamically by a function. It is a pragmatic solution that maintains type safety without requiring `any`.

## Utilities

Defined in `ProblemList.utils.ts`.

### `getDifficultyChipColor`

Pure function that returns the hex color corresponding to a given difficulty. Acts as a safe wrapper around `DIFFICULTY_COLOR_MAP` with a gray fallback for unexpected values.

```typescript
export const getDifficultyChipColor = (difficulty: ProblemDifficulty): string => {
  return DIFFICULTY_COLOR_MAP[difficulty] || '#8c8c8c';
};
```

**Usage examples:**

```typescript
getDifficultyChipColor('Easy');   // '#2db55d'
getDifficultyChipColor('Medium'); // '#feb800'
getDifficultyChipColor('Hard');   // '#ef4743'
```

Because it is typed with `ProblemDifficulty`, TypeScript prevents calls with invalid values at compile time. The `'#8c8c8c'` fallback only acts as a runtime safety net.

## Internal Hook

Defined in `ProblemList.hook.ts`.

### `useProblemList`

Hook that encapsulates the mobile drawer state. Exposes a minimal, clean API:

```typescript
export const useProblemList = () => {
  const [isMobileDrawerOpen, setIsMobileDrawerOpen] = useState(false);

  const handleToggleMobileDrawer = () => {
    setIsMobileDrawerOpen((prev) => !prev);
  };

  const handleCloseMobileDrawer = () => {
    setIsMobileDrawerOpen(false);
  };

  return {
    isMobileDrawerOpen,
    handleToggleMobileDrawer,
    handleCloseMobileDrawer,
  };
};
```

| Returned value | Type | Description |
|---|---|---|
| `isMobileDrawerOpen` | `boolean` | Current drawer open/closed state |
| `handleToggleMobileDrawer` | `() => void` | Toggles between open and closed |
| `handleCloseMobileDrawer` | `() => void` | Forces the drawer closed |

> **Current status:** The hook exists and is available, but the current version of `ProblemList.tsx` consumes it optionally. It was prepared for when the responsive mobile drawer variant is implemented (screens < 768px).

## Technical Decisions

### Why are `id` and `idNumber` separate fields?

- `id` is the database / system identifier. It is used for selection, React keys, and callbacks.
- `idNumber` is the visual number the user sees in the list (e.g. "1. Two Sum"). When the list is filtered, `id` might be `47` but `idNumber` would still be `1` according to the filtered order.

### Why a fixed width instead of `width: 100%`?

The Figma design defines this sidebar as a fixed-width panel within a three-column layout (list / editor / output). Using `width: 100%` would break the overall editor grid. If responsive behavior is needed, it should be handled from the parent layout.

### Why inline styles (`sx`) instead of `styled()` or CSS Modules?

Centralized `sx` objects in a dedicated `.styles.ts` file offer the best of both worlds: semantic organization and access to MUI's theming system, without needing additional styled components or managing class names. The separation into a dedicated file prevents `ProblemList.tsx` from becoming unreadable.

### Why `Record<string, unknown>` in `itemButtonStyles`?

MUI's `SxProps<Theme>` is a highly restrictive type that sometimes rejects nested CSS pseudo-elements (`&::before`, `&:hover`) when the object is built dynamically by a function. `Record<string, unknown>` is the safest typed alternative without resorting to `any`.

## Extension Guide

### Adding a new problem status

1. Add the new value to the type in `ProblemList.types.ts`:

```typescript
export type ProblemStatus = 'solved' | 'attempted' | 'unsolved' | 'skipped';
```

2. Add the corresponding icon in `ProblemList.constants.ts`:

```typescript
import SkipNextIcon from '@mui/icons-material/SkipNext';

export const STATUS_ICON_MAP = {
  solved: CheckCircleOutlineIcon,
  attempted: RemoveCircleOutlineIcon,
  unsolved: RadioButtonUncheckedIcon,
  skipped: SkipNextIcon, // new
} as const;
```

3. Add the color inline in `ProblemList.tsx`:

```typescript
const iconColor =
  problem.status === 'solved' ? '#2db55d' :
  problem.status === 'attempted' ? '#feb800' :
  problem.status === 'skipped' ? '#a855f7' :
  '#bcc1c4';
```

4. Update the mock in `ProblemList.test.tsx` if needed.

---

### Adding difficulty filters

The component does not currently include filters. To add them without breaking the existing interface, it is recommended to handle filtering in the parent component and pass the already-filtered array:

```tsx
// In the parent component
const [filter, setFilter] = useState<ProblemDifficulty | 'All'>('All');

const filteredProblems = filter === 'All'
  ? problems
  : problems.filter((p) => p.difficulty === filter);

<ProblemList
  problems={filteredProblems}
  selectedProblemId={selectedId}
  onSelectProblem={setSelectedId}
/>
```

### Implementing the mobile Drawer variant

The `useProblemList` hook already exposes the necessary state. Create a wrapper variant that wraps `ProblemList` inside an MUI `Drawer`:

```tsx
import { Drawer } from '@mui/material';
import { useProblemList } from './ProblemList.hook';

function ProblemListWithDrawer(props: ProblemListProps) {
  const { isMobileDrawerOpen, handleCloseMobileDrawer } = useProblemList();

  return (
    <Drawer open={isMobileDrawerOpen} onClose={handleCloseMobileDrawer}>
      <ProblemList {...props} />
    </Drawer>
  );
}
```
